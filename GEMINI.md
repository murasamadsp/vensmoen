# GEMINI.md - Kildesortering (Руководство по сортировке отходов)

Этот файл содержит инструкции и архитектурные решения для проекта **avfallssortering** — многоязычного руководства по сортировке отходов в Норвегии.

## Обзор проекта

- **Цель:** Простой, красивый и удобный для печати веб-сайт, объясняющий беженцам и новым жителям, как сортировать отходы дома.
- **Целевая аудитория:** Беженцы, иммигранты и новые жители Норвегии.
- **Технологии:** [Astro](https://astro.build) (статический вывод), без UI-фреймворков (Vanilla JS/CSS), Biome для линтинга и форматирования.
- **Данные:** Контент проверен на соответствие национальному стандарту **sortere.no**.

## Архитектура и важные файлы

### Интернационализация (i18n)

Проект использует ручное решение для i18n с маршрутизацией через `/[locale]/`.

- `src/i18n/locales.ts`: Определение поддерживаемых языков (код, название, направление текста/dir).
- `src/i18n/strings.ts`: Логика загрузки и определения типов для строк.
- `src/i18n/<код>.json`: Файлы переводов. **`nb.json` (норвежский) является основным источником истины.**
- `scripts/validate-i18n.mjs`: Скрипт валидации, проверяющий наличие всех ключей из `nb.json` в других языках.

### Данные

- `src/data/fractions.ts`: Содержит независимые от языка метаданные для каждой фракции отходов (например, `binColor`, `sources`, `lastVerified`).
- Текстовая информация для каждой фракции находится в `src/i18n/*.json` под ключом `fractions`.

### Страницы и компоненты

- `src/pages/[locale]/index.astro`: Основное руководство для конкретного языка.
- `src/pages/[locale]/quiz/index.astro`: Интерактивная викторина.
- `src/pages/[locale]/print/index.astro`: Упрощенный вид, оптимизированный для печати на A4 с QR-кодом.
- `src/components/FractionCard.astro`: Общий компонент, используемый как в веб-версии, так и в версии для печати (DRY).

## Разработка и команды

| Команда | Описание |
| :--- | :--- |
| `npm run dev` | Запуск локального сервера разработки. |
| `npm run build` | Сборка статического сайта в папку `dist/`. |
| `npm run check` | Валидация i18n, проверка типов и линтинг (рекомендуется перед push). |
| `npm run validate:i18n` | Проверка синхронизации всех переводов с норвежским оригиналом. |
| `npm run lint` | Запуск линтера и форматировщика Biome. |
| `npm run lint:write` | Автоматическое исправление ошибок линтинга. |

## Соглашения по разработке

1. **Источник истины:** Норвежский файл (`nb.json`) всегда является эталоном структуры и содержания переводов.
2. **Статика прежде всего:** Избегайте использования JavaScript во время выполнения (runtime), если это не абсолютно необходимо (например, для викторины).
3. **Дизайн:** Используйте существующие дизайн-токены в `src/styles/tokens.css`. Используйте системные шрифты.
4. **Верификация:** При обновлении контента обновляйте дату `lastVerified` в `src/data/fractions.ts` после проверки на `sortere.no`.
5. **Оптимизация для A4:** Все изменения в макете должны проверяться в режиме печати (`/[locale]/print/`), чтобы убедиться, что контент помещается на один лист A4.

## Как добавить новый язык

1. Создайте новый JSON-файл: `cp src/i18n/en.json src/i18n/<код>.json`.
2. Переведите значения в новом файле (ключи должны остаться идентичными).
3. Зарегистрируйте язык в `src/i18n/locales.ts` (добавьте в массив `locales`).
4. Импортируйте и добавьте новый словарь в `src/i18n/strings.ts`.
5. Выполните `npm run check` для проверки.
