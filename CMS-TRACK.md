## 🔴 Авто-перевод (главное — решает 2 боли разом)

- [ ] 🔴 **Авто-перевод изменённых полей.** Боль: правишь поле в одном языке — не знаешь, какие переводить в остальных 5. Решение: GitHub Action на пуш → находит изменённые поля (git diff) → переводит их через Claude API на остальные языки → коммитит обратно → деплоит. Также делает заполнение пустого гида тривиальным: пишешь текст в 1 языке → 5 остальных авто.
  - Нужно от пользователя: `ANTHROPIC_API_KEY` как GitHub Secret (стоимость — копейки на правку).
  - Поведение: изменённый язык = источник; перетирает то же поле в остальных переводом. (Манульные правки в других языках для этого поля затрутся — это и есть смысл.)
  - Встроить как шаг ПЕРЕД build в существующий `deploy.yml` (одна джоба, без гонки/рекурсии).
  - **СТАТУС:** скрипт `scripts/auto-translate.mjs` — **Claude-only** (Google убран, забраковали по качеству: «mottak»→«стійка реєстрації», «Fast»→«Виправлена»). Модель по умолч. `claude-sonnet-4-6`. Запуск: `npm run translate`. Режимы: `--fill-todo`, `--source X --paths ... --targets Y`, `--base <sha>`. Ключ из `.env` (в форке).
  - **РЕШЕНО (обновл.):** сотрудник нетехнический → ТОЛЬКО CMS, без терминала. Поэтому авто-перевод **вшит в `deploy.yml`** (best-effort). Умная система на ЛЮБОЙ язык: CI диффает → язык, который изменился = источник → переводит на остальные. Анти-луп: коммит бота через `GITHUB_TOKEN` не триггерит workflow. Анти-десинк: база диффа = последний коммит бота-переводчика (не прошлый пуш), так пропущенные из-за пустого баланса прогоны подхватываются позже, без вечной рассинхронизации. readonly НЕ ставим (правка любого языка должна работать).
  - **TODO юзер:** добавить `ANTHROPIC_API_KEY` как GitHub Secret (Settings → Secrets → Actions). Без него CI пропускает перевод (best-effort).
  - **БЛОКЕР:** на аккаунте Claude нет кредитов («credit balance too low»). Ключ валиден. Нужно пополнить → тогда гоним `uk → остальные` на ревью.
  - Анти-луп правила (на будущее, если авто): авто-перевод не становится источником (происхождение/хэш); ручную правку перевода не перетирать (флажок конфликта); Claude предлагает — человек подтверждает.
  - **STRUCTURED OUTPUTS:** включён (`output_config.format` json_schema, GA, без beta-хедера). Схема строится на лету из ключей (пути → string, `additionalProperties:false`) → гарантированно валидный JSON с теми же ключами, без чистки фенсов. Проверка `stop_reason` (refusal/max_tokens). max_tokens поднят до 8192. NB: schema варьируется per-call → кэш и так инвалидируется (но он и так был инертен <1024 ток).
  - **ПРОМПТ+КЭШ:** системный стиль-гайд (аудитория-беженцы, тон, сохранять `\n`/`•`/телефоны/email, не переводить имена собственные: Vensmoen, Saltdal, Røkland, UDI, NAV, Mental Helse, sortere.no) вынесен в `cache_control`-блок. ВАЖНО: кэш Sonnet 4.6 включается только при префиксе >1024 токенов — наши промпты мельче, так что на правках кэш не активен (и не нужен, цена и так доли цента). Cache-ready на случай массовых прогонов. Лог токенов кэша в выводе. Модель: `claude-sonnet-4-6` по умолч.; для ti/ar при желании `--model claude-opus-4-8` (дороже, но лучше). Цены/МТок: Sonnet $3/$15, Opus 4.8 $5/$25, кэш-чтение ×0.1.
