# AGENTS.md — Vensmoen mottak (інформаційний сайт центру)

Інструкції для AI-агентів і людей у проєкті **vensmoen** — багатомовному
статичному інформаційному сайті для мешканців центру прийому Vensmoen
(Рокланд, комуна Салтдал). Розділи: «Про центр», «Правила проживання»,
«Практичний гід» та повний гід із сортування відходів (`/waste/`).

## Огляд проєкту

- **Мета:** простий, охайний і придатний для друку сайт із постійною
  інформацією для мешканців центру.
- **Аудиторія:** біженці, іммігранти та нові мешканці Норвегії.
- **Технології:** [Astro](https://astro.build) (статичний вивід), без
  UI-фреймворків (vanilla JS/CSS), Biome для лінтингу та форматування.
- **Контент:** сортування перевірено за національним стандартом **sortere.no**.

## Залізні правила

1. **Не вигадуємо фактів.** Контент сайту — лише підтверджена інформація від
   адміністрації центру. Немає інформації → чесна заглушка
   (`site.comingSoon` або блок без `body`).
2. **`nb.json` — еталон.** Структуру та зміст перекладів завжди звіряємо з
   норвезьким файлом; решта мов слідують за ним.
3. **Жодного runtime-JS без потреби** (виняток — квіз).
4. **Дизайн-токени** з `src/styles/tokens.css`, системні шрифти.
5. **Однакова висота карток у всіх мовах:** довжина текстів різна, тому зони
   карток резервують фіксовану кількість рядків (див. `.emg__label`,
   `.contact-card__*`).
6. **Коментарі в коді — українською.**
7. **Друк:** зміни макета перевіряємо в `/[locale]/waste/print/` — все має
   вміщатися на один A4.

## Архітектура та важливі файли

### Інтернаціоналізація (i18n)

Ручне рішення з маршрутизацією через `/[locale]/`.

- `src/i18n/locales.ts` — список мов (код, назва, напрям тексту/dir).
- `src/i18n/strings.ts` — типізований лоадер словників.
- `src/i18n/<код>.json` — файли перекладів; **`nb.json` — джерело істини**.
- `scripts/validate-i18n.mjs` — перевірка синхронності ключів з `nb.json`.

### Дані

- `src/data/fractions.ts` — мовонезалежні метадані фракцій (`binColor`,
  `sources`, `lastVerified`).
- `src/data/emergency.ts` — національні екстрені номери + черговий телефон
  центру (`dutyPhone`).
- `src/data/location.ts` — посилання на карти; `src/data/transport.ts` —
  застосунки з розкладом транспорту.

### Сторінки та компоненти

- `src/pages/[locale]/index.astro` — головна: хаб із секційними картками,
  черговий телефон, екстрені номери.
- `src/pages/[locale]/about/` — про центр: офіс, контакти, карти, транспорт.
- `src/pages/[locale]/rules/` — правила проживання.
- `src/pages/[locale]/guide/` — практичний гід (контент частково заглушки).
- `src/pages/[locale]/waste/` — гід із сортування; `waste/quiz/` — квіз;
  `waste/print/` — A4-аркуш із QR-кодом.
- `src/components/InfoPage.astro` — спільний шаблон інфосторінок.
- `src/components/FractionCard.astro`, `GuideBody.astro` — спільні для веба
  та друку (DRY).

## Команди

| Команда | Опис |
| :--- | :--- |
| `npm run dev` | Локальний сервер розробки. |
| `npm run build` | Збірка статичного сайту в `dist/`. |
| `npm run check` | Валідація i18n + перевірка типів + лінтинг (обов'язково перед push). |
| `npm run validate:i18n` | Перевірка синхронності перекладів з `nb.json`. |
| `npm run lint` / `npm run lint:write` | Biome: перевірка / автовиправлення. |

## Як додати нову мову

1. `cp src/i18n/en.json src/i18n/<код>.json` і перекладіть значення
   (ключі не чіпати).
2. Імпортуйте файл у `src/i18n/strings.ts` і додайте в `dictionaries`.
3. Додайте рядок у `src/i18n/locales.ts` (`code`, `label`, `dir`,
   `htmlLang`); для RTL-скриптів — `dir: 'rtl'`.
4. `npm run check && npm run build` — валідатор зловить пропущені ключі,
   маршрути `/<код>/` створяться автоматично.

## Оновлення контенту сортування

Після звірки фракції з [sortere.no](https://sortere.no/) оновіть текст у
`src/i18n/*.json` (`fractions`) **і** дату `lastVerified` у
`src/data/fractions.ts`.
